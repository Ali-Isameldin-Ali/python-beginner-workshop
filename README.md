# Python Beginner Workshop Portfolio

Welcome to my Python Beginner Workshop repository! This README serves as the official documentation for my learning journey and projects over the course of the workshop.

## 📘 Week 1 Documentation: Core Python Fundamentals
**Main Task:** Guess the Number Game & Basic Data Operations
**Objective:** Understand basic syntax, control flow, and data structures.

**Concepts Applied & My Understanding:**
* **Variables & Data Types:** Learned how to store information using variables (Strings, Integers, Floats).
* **Operators & Math:** Used arithmetic operators (+, -, *, /) and comparison operators (==, <, >) to evaluate logic.
* **Control Flow (If/Else):** Applied decision-making logic to make programs interactive based on user input (e.g., checking if a guess is too high or too low).
* **Loops (For & While):** Used `while` loops to keep programs running until a condition is met, and `for` loops to iterate over a set number of times.
* **Data Structures (Lists & Dictionaries):** 
  * *Lists:* Stored multiple items in a single variable and accessed them using indexes.
  * *Dictionaries:* Stored data in key-value pairs, which is excellent for creating structured profiles (like a Student Profile).

## 📙 Week 2 Documentation: Functions & Object-Oriented Programming
**Main Task:** Interactive Bank System & Simple Budget Tracker
**Objective:** Learn code organization, reusability, and the principles of OOP.

**Concepts Applied & My Understanding:**
* **Functions (`def`):** Learned to group reusable code blocks together. This prevents code duplication and makes the main program much cleaner (Refactoring).
* **Classes & Objects:** 
  * A *Class* acts as a blueprint (e.g., `BankAccount` or `BudgetTracker`).
  * An *Object* is an instance of that class.
* **The `__init__` method & `self`:** Used the constructor to initialize object attributes when an object is created. The `self` keyword ensures that the object modifies its *own* specific data.
* **Class Methods:** Created functions inside classes (like `deposit()` or `withdraw()`) to define the behavior of the objects. 
* **Interactive CLI Systems:** Combined OOP with `while True` loops to create infinite menus that run until the user explicitly chooses to exit.

## 📗 Week 3 Documentation: Final Project
**Main Task:** Quiz Engine (Balanced Version)
**Concepts Applied:**
* Built a complete, interactive CLI application using a `QuizEngine` class.
* Used a List of Dictionaries as a question bank.
* Implemented the `random` module to shuffle questions.
* Tracked object state (score) and utilized methods for core game loops and a replay feature.
